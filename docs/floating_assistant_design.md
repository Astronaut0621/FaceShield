# FaceShield 桌面悬浮检测助手设计

## 设计目标

桌面悬浮检测助手面向电脑端视频通话场景，例如腾讯会议、微信视频、浏览器会议和远程面试。目标是在不侵入会议软件内部接口的前提下，对屏幕上的视频画面进行周期性截帧，并调用 FaceShield 后端完成 AI 换脸风险提示。

## 用户流程

1. 用户启动 FaceShield 后端。
2. 用户打开视频会议或视频通话窗口。
3. 用户启动桌面悬浮助手。
4. 用户登录 FaceShield 后端。
5. 用户框选对方视频画面区域。
6. 悬浮助手每隔固定秒数截取一帧。
7. 截帧图片提交至 `/api/detection/upload-and-detect`。
8. 后端完成上传留存、人脸裁剪、模型推理、风险等级映射和记录保存。
9. 悬浮助手展示风险等级、伪造概率、最近检测时间。

## 架构

```text
视频会议/通话窗口
        |
        | 屏幕区域截帧
        v
桌面悬浮助手（Tkinter）
        |
        | multipart/form-data
        v
FaceShield FastAPI 后端
        |
        | 人脸裁剪 + Paddle/mock 推理 + 记录留存
        v
悬浮窗风险提示
```

## 关键设计决策

### 不做腾讯会议内部插件

腾讯会议桌面端通常不开放稳定的视频流插件接口。直接接入会议软件内部视频流会带来兼容性、权限和平台限制。MVP 采用屏幕区域采样，可以覆盖腾讯会议、网页会议、微信视频等所有可见画面，更适合课程项目落地。

### 不在桌面端加载模型

桌面端只负责截图和显示结果，不直接加载 Paddle 模型。这样做可以避免在桌面助手中重复维护模型预处理、推理和 Grad-CAM 逻辑，所有算法能力仍统一由后端提供。

### 连续预警而非单帧强提醒

当前算法是图片级 real/fake 检测。视频场景通过连续截帧实现，因此单帧结果可能受表情、压缩、遮挡影响。助手记录最近 5 次检测结果，至少 3 次高风险时显示“连续预警”，用于降低单帧误报造成的干扰。

## 当前实现

实现目录：`desktop_assistant/`

- `floating_assistant.py`：主程序，包含悬浮 UI、区域选择、截图和 API 客户端。
- `requirements.txt`：桌面助手依赖。
- `run.ps1`：Windows 启动脚本。
- `smoke_test.py`：无 GUI、无真实后端的 API 客户端烟测。

## 验收标准

- 可以启动置顶悬浮窗。
- 可以登录 FaceShield 后端。
- 可以框选屏幕区域。
- 可以周期性截取所选区域。
- 可以将截帧提交到后端检测接口。
- 可以显示风险等级、伪造概率、最近检测时间。
- 后端检测记录仍正常留存。
- API 客户端烟测通过。

## 后续增强

- 在被选区域边缘显示半透明边框。
- 支持热力图弹窗预览。
- 增加系统托盘菜单。
- 增加自动定位会议窗口能力。
